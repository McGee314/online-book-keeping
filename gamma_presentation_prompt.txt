**Slide 6: Authentication Flow (Sequence Diagram)**
Show the Login flow step-by-step:
1. User submits username + password
2. Frontend POST /api/auth/login
3. Backend checks user exists, verifies BCrypt hash
4. Backend generates JWT (userId, username, expiry)
5. Response: { token, userInfo }
6. Frontend stores token in localStorage
7. Axios interceptor attaches "Authorization: Bearer <token>" to every request
8. JwtAuthInterceptor parses token → UserContext.setUserId()

**Slide 10: Key Technical Highlights**
Bullet points explaining the best engineering decisions:
1. **UserContext (ThreadLocal)** — isolates user data per request without passing userId everywhere; prevents data leaks between users
2. **Unified Result<T> wrapper** — every API response has { code, success, message, data } format for consistent error handling
3. **Axios interceptors** — request interceptor attaches JWT, response interceptor handles 401/403 globally
4. **Soft delete (MyBatis-Plus @TableLogic)** — data never truly deleted, recoverable by admin
5. **Docker Compose orchestration** — one command boots entire stack: database, backend, frontend
6. **Multi-currency support** — transactions stored in base currency, displayed in user-selected currency via exchange rates
7. **File upload validation** — server-side type & size checks, unique UUID filenames, persistent volume

---

**Slide 11: Error Handling Strategy**
- Backend: GlobalExceptionHandler (@RestControllerAdvice) catches all exceptions
  - BusinessException → 4xx with custom message
  - Validation errors → 400 with field-level messages
  - Unknown errors → 500 with "Internal server error"
- Frontend: Axios response interceptor
  - 401 → show server message (not hardcoded), redirect to /login (only if not on login page)
  - 403 → "Access denied"
  - Other errors → show server message or generic "Error {status}"
